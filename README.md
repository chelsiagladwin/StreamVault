  <h2><b>Overview</b></h2>

This project is a relational database designed for a movie streaming platform. It manages users, profiles, movies, TV shows, subscriptions, watch history, ratings, and categories while ensuring efficient storage and retrieval of data through normalization and SQL constraints
<h2>

<p>
  <h2><b>Features</b></h2>
  <center>
• User account and profile management <br>
• Storage of movies and TV shows<br>
• Genre and category classification<br>
• Subscription plan management<br>
• Watch history tracking<br>
• Content rating and reviews<br>
• Support for multiple profiles per user<br>
• Query optimization using joins, grouping, and aggregation functions<br>
<h2>
  </center>
<p>
  <h2><b>Database Schema</b></h2>
  <center>
The database consists of the following tables:
    1. Users – Stores user account information.<br>
    2. User_Profiles – Supports multiple profiles for each user.<br>
    3. Content – Contains details of movies and TV shows.<br>
    4. Categories – Stores genre information.<br>
    5. Content_Categories – Maps content to categories.<br>
    6. Subscription_Plans – Stores available subscription plans.<br>
    7. User_Subscriptions – Tracks user subscriptions.<br>
    8. Watch_History – Records viewing history and completion status.<br>
    9. Ratings – Stores ratings and reviews given by users.<br>
